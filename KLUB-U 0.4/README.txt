📥 УСТАНОВКА И ПЕРВЫЙ ЗАПУСК
═══════════════════════════════════════════════════════════════════════════

1️⃣  Замените старые файлы на новые:
    ├─ routes_data.py          (переписано)
    ├─ main_club.py            (обновлено)
    ├─ ui_club_v2.py           (обновлено)
    ├─ location_adapter_v2.py   (оставить)
    └─ run_club.bat            (оставить)

2️⃣  Убедитесь, что route.json имеет НОВУЮ структуру:
    ✓ "metadata" в корне
    ✓ "even_route" с beacons, signals, speed_limits, stations
    ✓ "odd_route" с тем же содержимым
    ✓ Каждый beacon содержит "meters"

3️⃣  Запустите приложение:
    
    Windows (рекомендуется):
      Double-click → run_club.bat
    
    Или через консоль:
      python main_club.py


🎮 ПЕРВОЕ ИСПОЛЬЗОВАНИЕ
═══════════════════════════════════════════════════════════════════════════

1. Откройте игру (MTA Province RP)
2. Запустите КЛУБ-У (run_club.bat или python main_club.py)
3. Откройте карту в игре (F11)

ℹ️  По умолчанию используются ЧЕТНЫЕ маршруты (направление 1)
    Чтобы переключиться на нечетные, нажмите Alt+2

🔍 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ
═══════════════════════════════════════════════════════════════════════════

❌ Приложение не запускается:
   → Проверьте наличие Python 3.7+
   → Запустите: python -m pip install -r requirements.txt
     (либо вручную установите PyQt5, pyautogui, psutil, keyboard)

❌ "GTA San Andreas не запущена":
   → Откройте MTA Province RP
   → Убедитесь, что процесс gta_sa.exe запущен

❌ "Иконка не найдена":
   → Откройте карту в игре (нажмите F11)
   → Убедитесь, что видна голубая квадратная иконка игрока (16х16)
   → Сохраните скриншот (Alt+S) для отладки

❌ "Координаты 0км 0пк" или неправильная позиция:
   → Проверьте, что route.json имеет правильную структуру
   → Убедитесь, что файл использует новый формат (с "even_route")
   → Включите отладку: в routes_data.py установите debug=True

❌ Горячие клавиши не работают:
   → Убедитесь, что окно Python имеет фокус
   → На некоторых системах нужны права администратора


🐛 ОТЛАДКА
═══════════════════════════════════════════════════════════════════════════

Чтобы включить подробное логирование в консоль:

1. В main_club.py найдите строку:
   routes_data.init_routes('route.json', debug=False, direction=1)

2. Измените на:
   routes_data.init_routes('route.json', debug=True, direction=1)
   
3. При каждом поиске координат будет выводиться:
   - Расстояния до ближайших маяков
   - Результаты интерполяции
   - Подробная информация о совпадениях

🚀 ДОПОЛНИТЕЛЬНЫЕ ОПЦИИ
═══════════════════════════════════════════════════════════════════════════

Переключение между маршрутами ПРОГРАММНО:
  
  import routes_data
  
  # Инициализируем с четными маршрутами
  routes_data.init_routes('route.json', debug=False, direction=1)
  
  # Позже переключаемся на нечетные
  routes_data.set_direction(2)
  
  # Получаем информацию о текущем маршруте
  info = routes_data.route_manager.get_location_info(x, y)
  print(f"Текущее направление: {info['direction']}")  # выведет 2


Поиск сигнала по названию:
  
  signal = routes_data.route_manager.get_signal_by_name("М21")
  if signal:
    print(f"{signal['name']} на {signal['meters']}м")
    print(f"Состояние: {signal['default_state']}")


Получение всех сигналов текущего маршрута:
  
  all_signals = routes_data.route_manager.get_all_signals()
  for signal in all_signals:
    print(f"{signal['name']}: {signal['meters']}м")

📝 ПРИМЕЧАНИЯ И СОВЕТЫ
═══════════════════════════════════════════════════════════════════════════

1. Направление автоматически не определяется, установите вручную (Alt+1/2)
2. Текущая скорость - пока что заглушка (0 км/ч), нужна интеграция спидометра
3. Скриншоты отладки сохраняются в текущую папку программы
4. В консоли выводится детальная информация (полезно при отладке)
5. Окно можно перемещать и изменять размер
6. Все коодинаты в соответствии с разрешением экрана 1920x1080
7. Если в sinal нет manual_state, поле будет пусто в UI
